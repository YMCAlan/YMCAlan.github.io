Const & Static & Extern 在C語言

Posted on 2023-12-28 Edited on 2024-01-04 In C

const Introduction

const 縮寫自 constant，代表 “恆定不變的”，是用來定義常量變數的關鍵字。

  • 被 const 修飾的變數不能更改，僅能進行讀取。
  • 被 const 修飾的函數參數在函數內不能被修改，這樣做作為一種保護機制，防止意外修改。不過，函數在使用時傳入的參數可以不是常量。

備註:
const int *p;解讀為p為指標指向常量int(指向常量的指標)，指標可更改，但指向物不行更改。
int* const p;解讀為p為常量指標指向int，指標不可改，但指向內容物可以改。
const int* const p;解讀為p為常量指標指向常量int，皆不可以改。
基本上解讀原則會在*左右兩邊
如果在星號左邊，表示某個靜態的資料型態，因此const int *p; == int const *p;這個是特例，都在星號左邊因此解讀上是一樣的。
如果在星號右邊，則表示指標是常量。

static Introduction

Static表示靜態，能夠修改變數的儲存區域和生命週期，

  • 靜態局部變數，在函數內被使用時只會初始化一次，可以維持某個函數中的狀態。

    • 作用域 : 僅限於宣告它的函數。
    • 在函數外不可見：即便是在同一檔案的其他函數中也無法訪問。

  • 靜態全局變數，限制其使用範圍，只能在其對應到的檔案使用，避免命名衝突。

    • 作用域：限定在宣告它的檔案（編譯單位）。
    • 檔案內可見：可以在檔案內的任何函數中訪問，但在其他檔案中不可見。

  • 靜態函數，與靜態變數一樣，只能在定義的檔案中使用，避免命名衝突。

    • 作用域：同樣也是限定在宣告它的檔案。
    • 檔案內可見：只能在同一檔案中的其他函數中呼叫，無法在其他檔案中被呼叫。

Extern Introduction

Extern允許被修飾變數在不同的source file中被使用。

extern "C"讓在c++中也可以使用c的函示庫，不會造成命名衝突。

Introduction

Array in C

輸入如下，我們就看到了這個矩陣佔20byte同時輸出其地址，而這個地址是陣列的第一個元素的地址

arr vs &arr

在以下的程式中，兩種輸出的東西，都是一樣的，但他們背後代表的資料型態是不一樣的。

首先使用指標方式，ptr為指標，指向的是int，因此我們可以將地址代入進去。
但是當對陣列使用&，會回傳指標並且指向的是一個陣列並且需要特定的長度，所以其形態為int(*)[5]，很明顯的看到這兩個東西雖然有相同一樣的數值，但所表現出來的東西是完全不同的。

ptr + 1 代表從 arr[0] -> arr[1]
而 &arr + 1 代表從 int()[5] -> int ()[5]
&arr + 1 位移了一整個陣列的大小。

上面式子最後回傳的是一個指標，為甚麼呢?首先我們先解讀&arr這個回傳了一個指標指向一個陣列且大小為5，接著我們依照*操作來依址取值
，&arr為指標指向一個陣列，所以裡面存放的是一個陣列的地址，所以取值後我們會得到是指標，陣列會隱性轉型成指向第一個陣列的指標喔。

泛型程式設計在C語言中

Posted on 2023-12-18 Edited on 2024-01-04 In C

Introduction

泛型(Generics)，是一種無形別的程式，能夠將相同的演算法應用於不同的資料型別上，那麼C由於是強型別語言，無法透過語法直接達成泛型的效果，但可以透過以下三種方式來達成

  • void* 指向void指標，使用強制轉型。
  • macro(巨集) 使用字元替換方式操作。
  • _Generic(C11)

Void*

我們可以透過void*在資料結構中進行強制轉型的動作，進而達到泛型效果，例如可以在一個linked list中指向不同型別的資料。
使用void方法就一定要用指標，讓實作時會比較複雜，但相對於macro來說是比較安全的。

Macro(巨集)是一種透過字串代換方式讓C語言能夠執行泛型操作，但比較難debug。

C語言中的變數宣告與解讀

Posted on 2023-12-16 Edited on 2024-01-04 In C

Introduction

解讀C語言的宣告式，平常看起來不太重要，但面對一些特殊的宣告式我們仍要有一個辦法去解讀他。

Right-left Rule

Right-left Rule指得是解讀C語言宣告式的一個法則。

符號

以下有3個基本的運算子，

  • *讀為 “指標指向的是” “as pointer to …” 只會在 (辨識字) 左邊出現
  • []讀為 “陣列元素為” “array of …” 只會在 (辨識字) 右邊出現
  • ()讀為 “函數回傳值” “function (parameter) returning” 只會在 (辨識字) 右邊出現

辨識子就是函數名稱或變數名稱喔

步驟

  1. 找到辨識字，解讀為 xxx是 或者 xxx is/ as
  2. 往辨識字的右邊看，如果看到
    • [] 解讀為 xxx是陣列，其元素為 xxx is the array of …
    • () 解讀為 xxx是函數, 其回傳值為 xxx is the function returning …
      直到遇到 單獨的 )
    • 如果遇到(則代表中間有其他東西要解讀，多半是遇到函數，要解讀其參數為何
  3. 接著往辨識字的左邊看
    • * 解讀為 指向的是， pointer to
    • datatype 就照念就好。

以下為不合法的C語言宣告

  • []() 沒有陣列中的元素是函數
  • ()() 沒有函數中回傳函數
  • ()[] 沒有函數回傳陣列
    但我們可以將其變成指標來使其合法

Example

int *ptr
declare ptr as pointer to int
宣告一個ptr指向整數型別的指標

int (*compare)(void*, void*)
declare compare as pointer to function (pointer to void, pointer to void) returning int
宣告一個compare的指標函數，其參數為(*void, *void),其回傳值為int

void **(*d) (int &, char **(*)(char *, char **))
declare d as pointer to function(ref to int, pointer to function (char*, char**) returning pointer to pointer to char) returning pointer to pointer to void.

int ptr[]()
不合法的宣告，宣告一個ptr為陣列其元素為函數，要修改為其元素為指標函數。
int (*ptr[])()
declare ptr as the array of pointer to function returning int

Introduction
